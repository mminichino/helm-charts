=== ON-PREMISES MODE ===

The initial admin API key will be written to /opt/initialAdminKey.txt on first startup.
{{ if .Values.postInstallJob.enabled }}
The post-install Job copies it into Secret {{ include "cs.initialAdminKeySecretName" . }} (data key {{ include "cs.initialAdminKeySecretKey" . }}):

  kubectl get secret -n {{ .Release.Namespace }} {{ include "cs.initialAdminKeySecretName" . }} -o jsonpath='{.data.{{ include "cs.initialAdminKeySecretKey" . }}}' | base64 -d
  echo

{{ end }}
To retrieve it manually from the admin pod:

  kubectl exec -n {{ .Release.Namespace }} \
    $(kubectl get pods -n {{ .Release.Namespace }} -l app.kubernetes.io/component=admin -o jsonpath='{.items[0].metadata.name}') \
    -- cat /opt/initialAdminKey.txt

Store this key securely - it will not be shown again!
{{ end }}

=== ADMIN SERVER ===

{{- if eq .Values.admin.service.type "NodePort" }}
Access the Admin API at:
  export NODE_IP=$(kubectl get nodes -o jsonpath='{.items[0].status.addresses[?(@.type=="InternalIP")].address}')
  export NODE_PORT={{ .Values.admin.service.nodePort }}
  echo "Admin API: http://$NODE_IP:$NODE_PORT"
{{- else if eq .Values.admin.service.type "LoadBalancer" }}
It may take a few minutes for the LoadBalancer IP to be available.
  kubectl get svc -n {{ .Release.Namespace }} {{ include "cs.fullname" . }}-admin -w
{{- else }}
Access the Admin API via port-forward:
  kubectl port-forward -n {{ .Release.Namespace }} svc/{{ include "cs.fullname" . }}-admin {{ .Values.admin.service.port }}:{{ .Values.admin.service.port }}
{{- end }}
